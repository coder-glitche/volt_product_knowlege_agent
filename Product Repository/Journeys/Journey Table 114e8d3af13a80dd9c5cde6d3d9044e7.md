## **1. Spreadsheet Structure Overview**

    - **Description:** The main phase of the user journey (e.g., User Acquisition and Onboarding, Eligibility and Limit Check).

    - **Description:** The specific sub-section within a phase (e.g., Launching the App, User Signup).

    - **Description:** A unique identifier for each user action for easy reference.

    - **Description:** Detailed description of the user action (e.g., "Launch the App", "Complete Signup").
5. **Number of Users**
    - **Description:** The total number of users who have performed this action within a specified time frame.
6. **Reason for Action**
    - **Description:** The primary motivation or purpose behind the user performing this action (e.g., "Access the platform", "Verify account").
7. **Data Source**
    - **Description:** The origin from where the data is collected (e.g., Google Analytics, Server Logs, User Surveys).
8. **Data Collection Method**
    - **Description:** How the data is being collected (e.g., Event Tracking, API Logs, Manual Entry).
9. **Timestamp**
    - **Description:** The date and time when the action was performed.
10. **Additional Notes**
    - **Description:** Any extra information or context related to the action (e.g., observed trends, anomalies).

## **2. Detailed Breakdown of Each Column**

### **A. Phase & Sub-Phase**

- **Purpose:** Categorize actions to understand where users are in their journey.
- **Tip:** Use the user journey map you provided to accurately assign phases and sub-phases.

### **B. Action ID**

- **Purpose:** Provides a unique reference for each action, making it easier to track and discuss.
- **Tip:** Follow a hierarchical numbering system based on your journey map (e.g., 1.2.1 for the first action in User Signup).

### **C. User Action**

- **Purpose:** Clearly describes what the user is doing.
- **Tip:** Use consistent terminology as outlined in your user journey map.

### **D. Number of Users**

- **Purpose:** Quantifies how many users perform each action, helping identify popular and less-used features.
- **Tip:** Decide on a consistent time frame for data collection (e.g., daily, weekly, monthly).

### **E. Reason for Action**
