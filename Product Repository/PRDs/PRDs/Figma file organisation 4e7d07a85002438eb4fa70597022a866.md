# **What problem are we solving?**

- Searching for updated files of features and different stages of the journey
- Get visibility on how each stage is handled for different lender
- Easy visibility on version history, compliance updates etc. done in the past - need to view in one place
- Allow storing secondary flows like: error handling, drop-off flows etc

# How to do measure success

1. For designers: 
    - Having a defined place to store in-progress, kick-off ready, explorations and archived design files under their respective charters
    - Having a defined checklist before design handovers
2. For non designers: 
    - Reducing number queries asked from non design folks to find a specific design file.

### Context

- Purpose of the Figma Org is not to see the flow but to document designs efficiently
    - To get a view on the journey, we will document the lender wise flow on another Figma file
        - Why? - Faster loading time for user journey file
        - Future scope: Will make a prototype of the journey

### File Structure

1. Cover
2. In-progress (in any)
3. Common flows
4. Main flows
5. Archive
    
    Version history will be maintained under archive segment is most cases, or below the latest designs wherever necessary
    
6. Explorations
    - A separate segment in the file to dump all rough designs and iterations

---

### Misc notes

- Most recent designs in the page get added upwards
- Secondary flows (error handling, empty states, loading etc.)
    - In Loan application journey - on the same page
    - In CORE: Servicing - separate page to be created
- For stages like Mandate (bajaj is last, tata is in Bank stage) - the designs will be stored in it’s own page, and for context the previous and next flows will be mentioned
- LAJ: Projects that require design changes in different parts of the journey:
    - A separate page will be created under Common Flows for dev handoff to document updates across the journey
    - Changes will also be updated in the main flow in their respective pages
- New designs will be created in drafts and then moved to respective files after review or under “In-progress” on the respective design figma file
