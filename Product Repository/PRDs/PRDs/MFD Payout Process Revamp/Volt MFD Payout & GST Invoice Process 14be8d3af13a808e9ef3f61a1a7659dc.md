# Volt MFD Payout & GST Invoice Process

## Overview

Volt provides payouts to its MFD partners. However, due to a lack of visibility into payout amounts, there are frequent support tickets. To reduce these, we are introducing:

- GST invoices generated by Volt.
- Updates to the payout statement.
- A structured process for MFD sign-off on invoices.

## MFD GST Invoice Flow

1. Calculate accurate base payouts.
2. Generate the GST invoice.
3. Send the invoice to the partner.
4. Obtain partner approval via email.
5. Process payments for approved invoices.
6. Address any reported issues.
7. Close GST for the month.

## **User Flows**

### **MFD Registration**

1. MFDs must register and provide:
    - Bank account details.
    - GSTN.
    - Submission via UI (partner dashboard) or email.
2. Verification Process:
    - Fetch bank details from the partner database.
    - If missing/invalid, trigger an email request for updates.
    - Verify via Penny Drop (avoid joint accounts).
    - Validate GSTN through [gov.in](https://services.gst.gov.in/services/searchtp).
    - Manually verify 140+ GSTNs and update records.

### **Payout Processing**

1. **Eligibility:**
    - MFDs receive payouts as per agreed terms.
    - GST-registered MFDs receive GST invoices.
    - Payouts above ₹15,000 incur TDS.
2. **Invoice Generation:**
    - Analytics generates payout and GST calculations.
    - Verifies bank accounts and GSTN.
    - Creates payout and GST invoices.
    - Updates ledgers accordingly.
    - Assists business in resolving partner queries.

### **Acknowledgment & Communication**

- Payout details are shared via email and dashboard (Phase 2).
- Email templates:
    - **Registration request** (if bank account/GSTN is missing).
    - **Payout confirmation request** (Google Form for issues).
    - **GSTN confirmation request** (Google Form for validation).

### **Reconciliation**

1. **Payout Status Scenarios:**
    - **Not processed** (bank/GSTN verification failed).
    - **Confirmed payout & GST invoice** (update ledger).
    - **Confirmed payout, GST invoice issues**:
        - Log issues in Google Sheets.
        - Resolve and update statuses.
        - If adjustments needed, follow issue-handling protocol.
        - Communicate updates via email.

## Overview

The Invoice Generator is a Python script that automates the generation of GST invoices in PDF format from CSV data. It supports bulk processing, payment status tracking, and generates detailed summaries of the invoice generation process.

## Prerequisites

- Python 3.7 or higher
- Pip package manager

Required Python packages:

```
pandas
num2words
weasyprint
Jinja2

```

## Installation

1. Create a virtual environment:

```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\\Scripts\\activate

```

1. Install required packages:

```bash
pip install pandas num2words weasyprint Jinja2

```

1. Clone or download the invoice generator script and place it in your project directory.

## File Structure

Your project directory should contain:

```
project_directory/
├── invoice_generator.py
├── invoice_template.html
├── invoice_data.csv
└── generated_invoices/     # Created automatically

```

## Usage

1. Prepare your input files:
    - Create `invoice_template.html` with your invoice design
    - Prepare `invoice_data.csv` with your invoice data
2. Run the script:

```bash
python invoice_generator.py

```

The script will:

- Process all entries in the CSV file
- Generate individual PDF invoices
- Create a summary CSV file
- Log all operations to `invoice_generator.log`

## Input Data Format

### CSV Fields

Required fields in `invoice_data.csv`:

- `Vendor_name`: Company/vendor name
- `STATE`: State of business
- `COUNTRY`: Country of business
- `GSTN`: GST registration number
- `Bank_name`: Bank name for payments
- `partner_bank_account`: Bank account number
- `BASE_AMOUNT`: Base amount before tax

Optional fields:

- `INVOICE_NO`: Custom invoice number (auto-generated if not provided)
- `INVOICE_DATE`: Invoice date (defaults to current date)
- `DUE_DATE`: Payment due date (defaults to invoice date + 30 days)
- `UTR`: Unique Transaction Reference for payments
- `PAYMENT_DATE`: Date of payment
- `PAID_AMOUNT`: Amount already paid
- `ADJUSTMENT`: Adjustment amount
- `partner_email`: Contact email
- `partner_phone`: Contact phone
- `partner_code`: Vendor code
- `pan_no`: PAN number

## Payment Status Logic

The system uses the following logic for payment status:

1. `PAID`: UTR exists and payment date is provided
2. `PENDING_VERIFICATION`: UTR exists but no payment date
3. `PENDING_PAYMENT`: UTR contains 'pending'
4. `UNPAID`: No UTR or invalid UTR

## Output

### Generated Files

1. Individual PDF invoices:
    - Named as: `{invoice_number}_{timestamp}.pdf`
    - Stored in `generated_invoices/` directory
2. Summary CSV file:
    - Named as: `invoice_summary_{timestamp}.csv`
    - Contains generation status and details for all invoices

### Summary CSV Fields

- Timestamp

- GST_Number
- PAN

## Customization

### Invoice Template

The HTML template supports:

- Custom styling with CSS
- Dynamic content using Jinja2 template variables
- Company logo and branding
- Custom layout and formatting

### Tax Calculation

Default GST rates:

- CGST: 9%
- SGST: 9%

To modify tax rates, update the `calculate_tax_amounts` method in the script.

### Invoice Numbering

Default format: `SLTR{fiscal_year}-{sequence}-{letters}`

- Fiscal year: Automatically determined
- Sequence: 3-digit random number
- Letters: 4 random uppercase letters

## Troubleshooting

Common issues and solutions:

1. Missing Fields Error:
    - Ensure all required fields are present in CSV
    - Check for empty or null values in required fields
2. GSTN Validation Error:
    - Verify GSTN format: `\\d{2}[A-Z]{5}\\d{4}[A-Z]{1}[A-Z\\d]{1}[Z]{1}[A-Z\\d]{1}`
    - Check for valid state code in first two digits
3. PDF Generation Issues:
    - Verify WeasyPrint installation
    - Check template HTML for syntax errors
    - Ensure all referenced assets (images, fonts) are available
4. Payment Status Issues:
    - Verify UTR format and presence
    - Check payment date format (YYYY-MM-DD)
    - Validate paid amount is numeric

For detailed error information, check `invoice_generator.log` in the project directory.
